<h2>Technische Dokumentation dieses Dokumentenservers</h2>
<h3>Dokumentation des Sicherheitskonzeptes</h3>
Auf unserem Dokumentenserver werden mehrere Sicherheitssysteme unterst&uuml;tzt.
<br/>
<ul>
<li>SSL</li>
Die gesamte Kommunikation zwischen Client und Dokumentenserver kann auf SSL-Basis &uuml;ber HTTPS ablaufen, wobei das entsprechende Zertifikat derzeit lediglich selbst erstellt ist und von keiner Certification Authority verifiziert wurde. Dennoch bewirkt die Kommunikation &uuml;ber HTTPS eine Verschl&uuml;sselung des Datenverkehrs, was eine sichere Verbindung bedeutet. Die Benutzung von HTTPS ist nicht vorgeschrieben (normales HTTP ist ebenfalls m&ouml;glich), aber wird empfohlen.

<li>Pr&uuml;fsummen</li>
F&uuml;r jedes Dokument, das &uuml;ber unseren Dokumentenserver publiziert wird, berechnen wir eine Pr&uuml;fsumme nach dem MD5-Algorithmus. Dies garantiert, dass das Dokument nach dem Upload weder von uns noch von Dritten modifiziert wurde. Die Berechnung passiert vollautomatisch und die Pr&uuml;fsumme wird in den dokumentbezogenen Metadaten abgespeichert.
<li>Digitale Signatur</li>
Der Nutzer hat die M&ouml;glichkeit, seine Dokumente mittels PGP oder GPG digital zu signieren. Auf diese Weise kann sichergestellt werden, dass das Dokument dem entspricht, das der Autor selbst erstellt hat und seine Identit&auml;t von der TUB garantiert werden kann. Der Schl&uuml;sselaustausch findet &uuml;ber den passwortgesch&uuml;tzten Zugang des Bibliotheksausweises statt, erst nach einem entsprechenden Login und einer Authentifizierung kann der Schl&uuml;ssel auf unseren Server &uuml;bertragen werden.

<br/>
Auch auf Bibliotheksseite werden die Dokumente beim Freischalten signiert. Dadurch wird (in Erg&auml;nzung zum Pr&uuml;fsummen-System) nachgewiesen, dass das Dokument seit seiner Freischaltung auf unserem Server nicht ge&auml;ndert wurde. Aus Gr&uuml;nden der Niedrigschwelligkeit ist die Signatur des Autors der eigenen Dokumente nicht zwingend notwendig.<br/>
</ul>

<h4>Software</h4>
Die Server sind mit dem Betriebssystem Linux ausgestattet. Zugrunde liegt ein Apache Webserver mit PHP-Unterst&uuml;tzung.
<p>
Die dem Dokumentenserver zugrundeliegende Software ist Opus, die seit 1997 an der Universit&auml;t Stuttgart entwickelt wird. Opus ist in PHP programmiert und als Backend wird eine MySQL-Datenbank benutzt. Das in der TUB eingesetzte System ist modifiziert und an die eigene Umgebung angepasst.
</p><p>
Anstatt der eigentlich in Opus eingesetzten Suchmaschine htdig wird Mnogosearch zur Volltextsuche genutzt. Zur Statistikerstellung werden eigene Scripte genutzt, die sich auf eine aus den Apache-Logdateien erstellte Datenbank st&uuml;tzen.

</p><p>
Die Abfrage von OAI-konformen Daten wird &uuml;ber die bei Opus mitgelieferte und im Haus angepasste OAI-PHP-Schnittstelle vorgenommen. Eine Erweiterung dieser Schnittstelle f√ºr XMetaDiss wurde an der TUB entwickelt.
</p><p>
Alle Dateien werden zur Sicherstellung der Authentizit&auml;t mit einer md5-Pr&uuml;fsumme gekennzeichnet und &uuml;ber einen GPG-Schl&uuml;ssel digital signiert (zumindest durch den freischaltenden Bibliothekar, ggf. auch durch den Autor selbst, sofern dieser dazu in der Lage ist). Eine entsprechende Erweiterung des Opus-Systems wurde ebenfalls an der TUB entwickelt.
